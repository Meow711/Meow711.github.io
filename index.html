ES6之let和const

Posted on 2017-04-19

一、let

let也是定义变量的一种声明方式，但是和var又有不同，主要记住下边4个注意点就行啦！

  • 统一块级作用域，不允许重复声明一个变量

  • 函数内不能用let重新声明函数的参数

  • 只在let命令所在代码块内有效

  • 用let声明变量没有变量提升的作用

二、const

var 和 let 是用来定义变量的，而 const 是用来定义常量的。常量，就是固定不变的量。

  • 用const定义一次后就不能再定义。

  • 也不能进行修改：

  • 只能在同一块级作用域中起作用

  • 不能只声明不赋值

  • const也可以声明一个对象

我的电脑在被我灌了牛奶后直接挂掉，开始重装系统、装软件、恢复以前的配置…没了固态硬盘，速度慢的一招回到解放前 :(

以前没有好的习惯，写过的代码，博客的配置文件都消亡了，今天又重搭博客，没想到又出了各种各样的问题。

  • github pages和自己的域名关联
    1）在自己的域名管理页面添加解析
    2）在自己的本地仓库中加入CNAME文件，文件内容写上自己的域名地址（如我的：javascript404.com），但这个也是最折磨我的问题。每次用hexo部署的时候，偏偏这个CNAME就像隐身了一样就是上传不上去。查了好久，其实都是hexo的问题，它会默认把md等文件转换成html, CNAME也会忽视不见。解决办法就是把READNE.md和CNAME放到source文件夹下就可以啦~

  • 之前更换主题总是不出效果
    1）可能的原因是浏览器的缓存，需要重启浏览器
    2）也可能是hexo出现了缓存，使用命令 hexo clean后再重新部署
